# Remaining Roadmap

## Launch-Critical

- Migrate frontend from JavaScript/JSX to TypeScript/TSX.
- Add full owner UI for editing pets, uploading multiple photos, managing documents and emergency contacts.
- Add `/pet/:id` route alias for public QR profile URLs.
- Implement password reset and account verification endpoints.
- Replace inline file storage with secure object storage.
- Add a durable notification worker.
- Add production-grade audit logs for all admin mutations.
- Complete administrator management screens.

## Role Portals

- Veterinary dashboard: practice profile, ownership verification workflow, pet detail verification and case history.
- Rescue dashboard: found-pet case management, active reunification tracking and reporting.
- Owner dashboard: sightings inbox, emergency contacts, documents, QR download and found/lost status actions.

## Partnerships And Donations

- Add optional monthly supporter records without feature gating.
- Add corporate sponsor records.
- Add veterinary sponsor records.
- Add animal charity partnership records.
- Keep all platform functionality free.

## Quality

- Convert tests to isolated database fixtures.
- Add frontend component tests.
- Add API contract tests.
- Add accessibility testing.
- Replace temporary stock photography with licensed UK pet welfare imagery.
